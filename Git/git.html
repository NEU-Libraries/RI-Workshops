Command-line interface (CLI)

The user provides the computer a command. The computer reads it, executes it, and prints its output.

Git is most commonly used via the CLI

Getting Started

Let's

  • Move to the desktop
  • Make a directory called "planets"
  • Move into planets
cd ~/Desktop 
mkdir planets 
cd planets

Create a new repository

git init

What happened?

ls -a

What's our status

git status

Check our location

pwd

Create mars.txt

nano mars.txt

Type "Cold and dry, but everything is my favorite color"

Leaving Nano

^o (control-o) - write out the file

^x (control-x - exit, will prompt to save)

Mars.txt

ls

and 

cat mars.txt

Check our status

git status

Adding files

git add mars.txt

Commit our file

git status

and 

git commit -m "Start notes on Mars as a base"

Check our status

git status

git log

git log

adding more text

nano mars.txt

Add "The two moons may be a problem for Wolfman"

git diff

git diff

trying to commit

git commit -m "add concerns about Mars' moons on Wolfman"

staging

Allows us to select files to add and commit together in logical stages 

git add mars.txt

and then 

git commit -m "add concerns about Mars' moons on Wolfman"

add more text

nano mars.txt

Add "But the Mummy will appreciate the lack of humidity"

git add

git add mars.txt

git diff and staging

git diff

and then 

git diff --staged

finish with 

git commit -m "add Mummy's concerns"

Logging options

git log --oneline --graph --all --decorate

mkdir results

and then

results/

and …
